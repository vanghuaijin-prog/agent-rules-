# 诉讼案件统一流程 v1.0

> 诉讼案件的唯一入口。无论"写文书"还是"整理证据"，都从这里走。内部分轻量/完整两档，按证据类别数自动分流。
>
> **详细操作规范引用：** 02-01（证据整理SOP-v2.1）、02-02（文书写作SOP）。本文件是路由层，不重复定义细节。

## 1. 流程总览

```
案件进入（Inbox，文件夹名含"1"，有MD指令）
│
├── Phase 1：材料消化与证据整理（必经）
│
├── Phase 2：法律研究（norm check + case check）
│
├── Phase 3：案件分析与文书撰写
│
├── Phase 4：PDF合订本（仅完整模式）
│
└── Phase 5：收尾（自检 + 生成Word + 沉淀）
```

## 2. 分流规则

| 条件 | 模式 | 跳过 |
|------|------|------|
| 证据类别 ≤ 8 类 | **轻量模式** | Phase 4（不做证明组合订本），映射表简化为四列 |
| 证据类别 > 8 类 | **完整模式** | 无跳过，全Phase执行 |

**判断时机：** Phase 1 分类完成后确定模式。

## 3. Phase 1：材料消化与证据整理（必经）

> 无论什么案件，证据整理是第一步。不整理证据就写文书 = 盲写。

## 8. 文件命名

### 8.1 项目级（文件夹名）

`{ID前缀}-{YYMMDD}-{序号}-{ShortName}`

示例：`S-260624-001-陈玉欣诉周万略健康权纠纷`

### 8.2 文件级（客户可见的交付文件）

`{案名简称}-{文书类型}-v{N}.docx`

示例：
- `陈玉欣诉周万略-民事起诉状-v1.docx`
- `陈玉欣诉周万略-证据清单-v1.docx`
- `陈玉欣诉周万略-证据合订本-v1.pdf`

**不写客户简称、不写日期、不用｜分隔符。** 日期和案号信息已在项目级ID中。

### 8.3 内部工作文件

`{ID前缀或编号}-{描述}-v{N}.{ext}`

示例：
- `M-00-证据映射表.xlsx`
- `L-01-01-民事起诉状-v2.md`

## 9. 检查清单

### 轻量模式（≤8类证据）

- [ ] 材料已全部读取并OCR
- [ ] P1/D1/L/M/Q结构已建
- [ ] 证据已分类编号重命名
- [ ] 四列映射表已建
- [ ] norm check + case check 已完成
- [ ] 案件分析五要素已完成
- [ ] Markdown文书已撰写
- [ ] 证据清单（六列）已整理
- [ ] 自检通过
- [ ] Word已生成
- [ ] 已移入Active/

### 完整模式（>8类证据）

- [ ] 以上全部
- [ ] 证明组已设计（从诉求倒推）
- [ ] 映射表七列完整
- [ ] 关键证据已标记
- [ ] 金额比对已通过
- [ ] PDF合订本已生成并验证
- [ ] 证据清单页码已回填
- [ ] 映射表三处同步无遗漏
