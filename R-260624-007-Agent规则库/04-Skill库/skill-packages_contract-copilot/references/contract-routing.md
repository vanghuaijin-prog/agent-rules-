# contract-routing

---

## 一、使用规则

1. 先按交易实质识别合同类型，再按本清单归入 12 类目录。
2. 未单列的细分类型，优先复用同类主文件，不空转为“未覆盖”。
3. 输出审查结论时必须包含：风险等级、触发条件、可执行整改动作。
4. 起草时必须先输出：主合同类型、配套协议类型、待补事实清单、条款骨架、程序性前提。
5. 对混合型交易，采用“主合同类型 + 配套协议类型”的双标签方式。
6. 起草进入正文前，先产出一张“起草路由卡”：主文件、推荐交付包型、必带附件、优先骨架顺序、待确认问题。

## 三、起草路由速查

| 类型 | 起草时优先确认 | 必补事实 | 推荐交付包型 | 常见配套 / 双标签 |
|:---|:---|:---|:---|:---|
| 买卖合同 | 标的、交付、验收、价款 | 标的权属/规格/数量、价款税费、交付验收、风险转移、瑕疵救济 | 完整买卖合同 / 框架协议 + 订单 / 主合同 + 安装调试附件 | 担保、房地产程序、服务配套 |
| 租赁合同 | 租赁物、用途、租期、返还 | 权属与现状、租期用途、租金押金、维修改造、安全责任、返还条件 | 完整租赁合同 / 主合同 + 交接清单 / 主合同 + 管理规则 | 房地产程序、设备安全、带机/带服务安排 |
| 服务类合同 | 服务边界、成果、配合、验收 | 服务清单、交付件、配合义务、费用结构、知识产权、数据边界、退出交接 | 主合同 + SOW/服务清单 / 框架服务协议 + 任务单 | 房地产配套、知识产权、劳动边界 |
| 知识产权类合同 | 权利链、授权/转让范围、成果归属 | 权利基础、地域期限、许可范围、源文件/样品交付、登记备案、侵权处置 | 许可/转让主合同 + 授权范围表 / 开发主合同 + 成果交付表 | 服务合同、技术开发、数据合规 |
| 担保类合同 | 主债权、担保范围、实现路径 | 主债权金额/期限、担保财产状态、登记或控制、违约触发、处置顺序、反担保 | 主债权合同 + 担保合同 / 担保合同 + 反担保文本 | 借款、买卖、投资安排 |
| 借贷与赠与合同 | 金额、期限、交付凭证、返还机制 | 本金/利率/用途、借款交付、到期与提前到期、担保、撤销或返还条件 | 借款合同 + 担保文本 / 赠与合同 + 财产交付清单 | 担保、公司投资、赠与附义务 |
| 互联网协议 | 产品规则、用户边界、数据处理 | 服务范围、账号规则、数据处理、SLA、版本更新、通知方式、内容处置 | 用户协议 + 隐私政策 + 订单协议 / ToB 主协议 + SLA + 数据处理附件 | 隐私政策、订单协议、用户协议联动 |
| 婚姻家事类合同 | 身份关系、财产范围、生效程序 | 财产清单、债务承担、抚养/扶养安排、证照材料、生效条件、登记或确认程序 | 主协议 + 财产清单 / 离婚协议 + 子女抚养安排 | 房地产、赠与、继承安排 |
| 劳动用工类合同 | 用工模式、报酬、考勤、退出 | 用工类型、岗位职责、期限报酬、社保考勤、成果归属、保密竞业、解除交接 | 主合同 + 保密/竞业/培训附件 / 外包主合同 + 服务清单 | 保密、培训服务期、公司治理 |
| 房地产类合同 | 权属状态、程序、交付过户 | 土地/房屋状态、审批或备案、价款税费、交付过户、开发建设义务、违约退出 | 主合同 + 权属/审批附件 / 主合同 + 联建或代建配套协议 | 买卖、服务、建设工程、公司投资 |
| 建设工程类合同 | 工程范围、工期、价款、质量 | 图纸/清单、工期节点、进度款结算、变更签证、质量安全、验收保修、索赔机制 | 主合同 + 图纸/工程量清单 / 总承包主协议 + 分包配套 | 分包、监理、设计、房地产开发 |
| 公司投资类合同 | 交易结构、估值、治理、交割 | 投前投后口径、先决条件、交割文件、陈述保证与披露、治理权、退出机制 | 主协议 + 股东协议 + 章程修订稿 + 披露函 + 交割清单 | 股东协议、对赌、股权转让、资产收购 |

## 五、起草骨架优先顺序

| 类型 | 先写哪几组条款 | 最易遗漏的程序 / 退出 / 联动 |
|:---|:---|:---|
| 买卖合同 | 标的 -> 交付验收 -> 价款税费 -> 风险/所有权 -> 售后违约 | 质量异议期、替代采购权、保留所有权、安装调试接口 |
| 租赁合同 | 租赁物现状 -> 租期用途 -> 租金押金 -> 维修改造 -> 返还违约 | 押金抵扣逻辑、消防/装修审批、提前解约、转租管理 |
| 服务类合同 | 服务范围 -> 交付件/验收 -> 配合顺延 -> 费用结算 -> 成果归属 -> 解除移交 | 增项机制、数据边界、既有模板权利、退出交接 |
| 知识产权类合同 | 权利基础 -> 授权/转让范围 -> 成果交付 -> 费用 -> 侵权处置 | 背景知识产权许可、登记备案、第三方素材合规 |
| 担保类合同 | 主债权 -> 担保范围 -> 担保物状态 -> 生效与设立 -> 实现路径 | 登记/控制手续、反担保、处置顺序、主合同联动 |
| 借贷与赠与合同 | 本金/标的 -> 交付凭证 -> 利率或条件 -> 到期返还 -> 违约救济 | 提前到期、放款凭证、担保联动、撤销条件 |

## 六、未单列类型处理办法

对尚未拆分为独立文件的细分合同，统一按以下顺序处理：

1. 进入本清单定位到 12 类与主文件。
2. 先输出一张“起草路由卡”，至少写明：主合同类型、主文件、推荐交付包型、配套协议类型、必带附件、待确认问题。
3. 再在 `templates/合同起草信息清单.md` 中记录待补事实、程序性前提和骨架顺序。
4. 从对应主文件提取条款级风险框架或条款骨架。
5. 回到 `review-framework.md` 复核宏观/中观/微观结构性风险是否遗漏。
6. 审查场景下，在报告中说明“本次按同类主文件审查，后续可拆分为独立模板”；起草场景下，在草案说明中标注“本次按同类主文件起草，未提及事项待补充”。

## 七、输出一致性要求

### 1) 审查输出

- 输出字段：风险名称、风险等级、风险后果、判别标准、推荐措辞、风险示例、法律依据、整改建议、相关条款。
- 风险等级：P0（签前必改）/ P1（优先改）/ P2（优化项）。
- 结论格式：可签 / 有条件可签 / 不建议签 + 先决事项清单。

### 2) 起草输出

- 先输出“起草路由卡”：主合同类型、配套协议类型、主文件、推荐交付包型、必带附件、程序性前提、待确认问题。
- 再输出：待补事实、条款骨架、推荐措辞、交付物。
- 缺失信息统一标注：`未提及/待补充`。
- 交付格式：合同草案 / 条款清单 / 待确认问题表 / 文件包清单。
