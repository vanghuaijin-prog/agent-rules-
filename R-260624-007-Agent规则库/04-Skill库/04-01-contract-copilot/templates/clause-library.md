# 合同条款库

本条款库提供常用合同条款的推荐措辞，供审查和起草合同时参考使用。

本文件定位为“起草支撑层”，不是独立于 `references/` 之外的另一套知识体系。起草时应先使用：

1. `references/review-framework.md` 确认结构性条款骨架；
2. `references/contract-routing.md` 定位合同类型；
3. `templates/合同起草信息清单.md` 收集主合同类型、配套协议类型和待补事实；
4. 对应 `references/XX-合同类型/` 主文件确认该类合同的专属条款和风险点；
5. 最后再从本条款库抽取可复用的通用候选措辞。

后续条款库的扩充，优先从 12 类合同主文件中反向抽取高频共性条款，而不是平行维护另一套“起草 reference”。
本条款库只负责提供可复用措辞，不负责替代映射清单和主文件去确定合同骨架。

## 一、通用条款

### 1.1 陈述与保证条款

```
第X条 陈述与保证

甲方/乙方在此向对方陈述并保证如下:

1. 主体资格: 其是依法设立并有效存续的【有限责任公司/股份有限公司】,具有签署和履行本合同的全部法定资格和权利能力。

2. 授权与批准: 其签署和履行本合同已获得其内部所有必要的授权和批准,不违反其公司章程、内部规章制度或其与任何第三方签订的任何协议,不需要取得任何第三方的同意或政府部门的审批。

3. 合同效力: 本合同一经签署即对其构成合法、有效、具有约束力的义务。

4. 无诉讼: 其目前不存在任何可能对其履行本合同义务产生重大不利影响的诉讼、仲裁、行政处罚或其他法律程序。

5. 【根据具体合同类型增加特定的陈述与保证,如所有权保证、知识产权保证等】
```

### 1.2 保密条款

```
第X条 保密义务

1. 保密信息的定义: 本合同所称"保密信息"是指一方(披露方)向另一方(接收方)披露的、标注为"保密"或根据其性质和披露情境应被合理理解为保密的任何信息,包括但不限于技术信息、商业信息、财务信息、客户信息等。

2. 保密义务: 接收方应对保密信息承担严格的保密义务,不得向任何第三方披露、泄露或传播保密信息,不得将保密信息用于本合同目的之外的任何其他用途。

3. 例外情况: 以下信息不属于保密信息:
   (1) 在披露时已为公众所知悉的信息;
   (2) 在披露后非因接收方原因而成为公众所知悉的信息;
   (3) 接收方从有权披露该信息的第三方合法获得的信息;
   (4) 接收方独立开发的、未使用披露方保密信息的信息。

4. 保密期限: 接收方的保密义务自其收到保密信息之日起生效,至本合同终止后【三】年内持续有效。

5. 法律要求披露: 如接收方因法律、法规或政府部门的要求需要披露保密信息,应在披露前及时通知披露方,并尽合理努力协助披露方寻求保护令或其他保护措施。
```

### 1.3 违约责任条款

```
第X条 违约责任

1. 一般违约责任: 任何一方未能按照本合同约定履行其义务或其陈述与保证不真实、不准确的,应承担违约责任,并赔偿因此给守约方造成的全部损失。

2. 逾期付款: 如买方/委托方未按约定时间支付款项,每逾期一日,应按逾期金额的【0.5】‰向卖方/服务方支付违约金。

3. 逾期交付/履行: 如卖方/服务方未按约定时间交付标的物/完成服务,每逾期一日,应按合同总价款的【0.5】‰向买方/委托方支付违约金,但该等违约金总额不超过合同总价款的【10】%。

4. 根本违约: 如一方发生以下情形之一,构成根本违约,守约方有权单方解除合同,并要求违约方赔偿全部损失:
   (1) 逾期交付/付款超过【30】日的;
   (2) 明确表示或以其行为表明不履行合同主要义务的;
   (3) 【其他根本违约情形】

5. 违约金与赔偿的关系: 违约金不足以弥补守约方实际损失的,守约方有权要求违约方赔偿超出部分的损失。
```

### 1.4 不可抗力条款

```
第X条 不可抗力

1. 定义: 本合同所称"不可抗力"是指不能预见、不能避免且不能克服的客观情况,包括但不限于地震、台风、洪水、火灾、战争、罢工、政府行为、法律法规变更等。

2. 通知义务: 遭受不可抗力的一方应在不可抗力发生后【3】日内书面通知对方,并在【15】日内提供相关证明文件(如气象部门、公证机关或政府部门出具的证明)。

3. 法律后果: 
   (1) 因不可抗力导致一方部分或全部不能履行合同义务的,根据不可抗力的影响,该方可部分或全部免除责任。
   (2) 如不可抗力持续超过【30】日,任何一方均有权以书面形式通知对方解除本合同,双方互不承担违约责任。

4. 减轻损失义务: 遭受不可抗力的一方应采取一切合理措施,尽量减轻不可抗力对合同履行的影响。
```

### 1.5 争议解决条款

#### 选项一: 诉讼

```
第X条 争议解决

因本合同引起的或与本合同有关的任何争议,双方应首先通过友好协商解决。协商不成的,任何一方均有权向【被告住所地/合同履行地/合同签订地/原告住所地】人民法院提起诉讼。
```

#### 选项二: 仲裁

```
第X条 争议解决

因本合同引起的或与本合同有关的任何争议,双方应首先通过友好协商解决。协商不成的,任何一方均可将争议提交【北京仲裁委员会/中国国际经济贸易仲裁委员会】,按照申请仲裁时该会现行有效的仲裁规则进行仲裁。仲裁地点为【北京】。仲裁裁决是终局的,对双方均有约束力。
```

### 1.6 合同解除条款

```
第X条 合同解除

1. 协商解除: 经双方协商一致,可以书面形式解除本合同。

2. 单方解除: 如发生以下情形之一,守约方有权以书面形式通知对方解除本合同:
   (1) 对方发生根本违约的;
   (2) 对方丧失履约能力的(如被吊销营业执照、宣告破产、进入清算程序等);
   (3) 【其他约定的解除情形】

3. 解除后果: 合同解除后,尚未履行的,终止履行;已经履行的,根据履行情况和合同性质,当事人可以请求恢复原状或者采取其他补救措施,并有权请求赔偿损失。

4. 存续条款: 合同解除不影响以下条款的效力: 保密条款、知识产权条款、争议解决条款以及根据其性质应当存续的其他条款。
```

### 1.7 合同文件组成与优先顺序条款

#### 适用场景: 买卖、服务、技术开发、软件许可、建设工程、投资交易等

```
第X条 合同文件组成与优先顺序

1. 本合同正文、补充协议、订单/任务书/工作说明书、报价单、技术规范、图纸清单、交付/交割清单以及双方书面确认的其他文件，共同构成本合同完整组成部分。

2. 同一事项存在不一致的，除双方另有明确约定外，适用顺序如下：
   (1) 双方最后签署的补充协议；
   (2) 本合同正文；
   (3) 经双方确认的专项附件或专项协议；
   (4) 订单、报价单、排期表等执行性文件。

3. 平台规则、第三方模板或示范文本需一并适用的，以双方书面确认且不与本合同正文冲突为限。

4. 各附件、订单和清单应标注版本号或确认日期；未标注的，以双方最后一次书面确认版本为准。
```

### 1.8 交付、验收与异议期条款

#### 适用场景: 买卖、服务、技术开发、承揽、建设工程、外包等

```
第X条 交付、验收与异议期

1. 交付方应按本合同约定提交成果、货物、文件或工作内容，并同步提交交付清单、测试记录、验收资料或其他必要证明材料。

2. 接收方应于收到交付物后【X】日内完成外观、数量或形式验收，并于【X】日内完成功能、质量或实质验收；逾期未提出书面异议的，视为相应阶段验收通过。

3. 接收方认为交付物不符合约定的，应一次性书面列明问题清单、依据和整改要求；交付方应于【X】日内完成整改并重新提交验收。

4. 对于需要试运行、试用、观察期或阶段评审的项目，双方应形成书面记录，并以该记录作为最终验收或结算依据之一。
```

### 1.9 配合义务与顺延条款

#### 适用场景: 服务、技术开发、建设工程、投资交割、租赁等

```
第X条 配合义务与顺延

1. 【委托方/发包人/买方/投资方】应按约提供资料、接口人、场地条件、审批协助、确认意见及其他履约所需配合。

2. 因一方未按约履行配合义务，导致对方履约延误、成本增加或阶段目标无法实现的，相关期限应相应顺延，增加的合理成本由责任方承担。

3. 一方提交确认、审批或反馈事项后，对方应于【X】日内书面回复；逾期未回复的，双方按本合同约定的默认处理规则执行。

4. 因配合迟延导致合同目的明显无法实现的，守约方有权要求整改、暂停履行或按约解除合同。
```

### 1.10 条件成就、程序前提与长停条款

#### 适用场景: 投资、股权/资产交易、房地产、担保、监管要求较强的交易

```
第X条 条件成就、程序前提与长停

1. 本合同生效、付款、交割、过户、放款或其他关键义务的履行，以附件《先决条件清单》所列事项完成为前提，包括但不限于内部决议、审批、备案、登记、担保设立、资料交付等。

2. 各项先决条件应由约定责任主体在【X】日前完成，并向对方提交相应证明文件。

3. 截至【长停日】先决条件仍未完成的，守约方有权选择：
   (1) 书面同意延期；
   (2) 暂停履行后续义务；
   (3) 解除本合同，并按已履行情况办理返还、结算和损失分担。

4. 任一方不得无正当理由阻碍先决条件成就；因其原因导致长期未成就的，应承担相应违约责任。
```

### 1.11 通知送达条款

#### 适用场景: 全类型合同，尤其借贷、担保、租赁、劳动、投资争议场景

```
第X条 通知与送达

1. 双方确认以下地址、电子邮箱和联系电话为有效送达信息；任一方变更的，应于变更后【3】日内书面通知对方。

2. 通过专人递送、快递、挂号信、电子邮件或双方约定的电子系统发送的通知，分别以下列时间视为送达：
   (1) 专人递送的，以签收之日为送达日；
   (2) 快递或挂号信的，以签收日或退回日后的【第X日】为送达日；
   (3) 电子邮件或系统消息的，以发送成功并可合理读取之日为送达日。

3. 一方未按约更新送达信息导致通知退回、拒收、无法投递的，不影响送达效力。

4. 催收、违约、解除、交割、验收异议等重要通知，均适用本条。
```

### 1.12 责任限制及例外条款

#### 适用场景: 服务、软件许可、技术开发、平台协议等

```
第X条 责任限制及例外

1. 除本条第2款另有约定外，任何一方因履行本合同承担的累计赔偿责任，以【合同总价款/已支付费用的X倍】为上限。

2. 前述责任限制不适用于以下情形：
   (1) 故意或重大过失；
   (2) 保密义务违反；
   (3) 知识产权侵权；
   (4) 数据泄露、非法处理数据或信息安全事件；
   (5) 人身损害或依法不得免责的其他情形。

3. 守约方为实现债权发生的合理费用，包括律师费、保全费、鉴定费、差旅费等，可由违约方承担。
```

## 二、专项条款

### 2.1 知识产权归属与背景技术条款

#### 适用场景: 技术开发、委托创作、服务合同等

```
第X条 知识产权

1. 背景知识产权: 双方在签订本合同前已经合法拥有的技术、代码、模板、工艺、数据、文档及其他成果，仍归原权利人所有，不因履行本合同当然转移。

2. 定制成果归属: 为履行本合同专门形成并由【甲方/委托方】支付对价取得的定制成果，其知识产权或约定范围内的独占使用权归【甲方/委托方】享有，具体范围以附件《交付清单》和《成果归属说明》为准。

3. 背景技术许可: 为实现定制成果正常使用之目的，乙方(服务方/受托方)应向甲方授予其背景技术在合同目的范围内的必要使用许可；未经乙方书面同意，甲方不得超出约定范围单独转让、许可或拆分使用。

4. 第三方素材和开源组件: 一方引入第三方素材、软件、代码、字体、图片、音乐或开源组件的，应保证授权完整、范围明确，并按约提供授权或合规证明。

5. 侵权责任: 如任何工作成果或交付物侵犯第三方知识产权，由责任方负责处理纠纷、承担赔偿责任，并使守约方免受损失。
```

### 2.2 服务水平协议 (SLA) 条款

#### 适用场景: SaaS、云服务、IT运维等

```
第X条 服务水平协议

1. 服务可用性: 甲方(服务商)承诺,本服务的月度可用性不低于【99.9】%。

2. 可用性计算公式:
   月度可用性 = (月度总分钟数 - 服务不可用分钟数) / 月度总分钟数 × 100%

3. 服务不可用的定义: 当乙方(客户)无法通过正常方式访问和使用本服务,且该情况持续超过【5】分钟的,视为服务不可用。

4. 排除情况: 以下情况不计入服务不可用时间:
   (1) 甲方提前【72】小时通知乙方的计划内系统维护;
   (2) 因乙方自身原因(如网络故障、设备故障)导致的不可用;
   (3) 因不可抗力导致的不可用;
   (4) 因第三方服务(如电信运营商)故障导致的不可用。

5. 赔偿标准: 如因甲方原因导致服务月度可用性低于承诺标准,则按以下标准进行赔偿:
   - [99.0% - 99.9%): 赔偿当月服务费的【10】%
   - [95.0% - 99.0%): 赔偿当月服务费的【25】%
   - [低于95.0%]: 赔偿当月服务费的【50】%

6. 赔偿方式: 赔偿以服务费减免或延长服务期限的方式进行,不以现金形式支付。
```

### 2.3 数据所有权与安全条款

#### 适用场景: SaaS、云服务、数据处理等

```
第X条 数据所有权与安全

1. 数据所有权: 双方确认,乙方(客户)在使用本服务期间上传、存储、处理的任何数据(下称"客户数据")的全部权利、所有权和利益,均归乙方所有。甲方(服务商)除为履行本合同之目的外,不得以任何形式使用、复制、披露或传播客户数据。

2. 数据安全措施: 甲方应采取不低于行业通行标准的物理、技术和管理安全措施,保护客户数据的安全和保密,防止数据泄露、毁损、丢失或被非法访问、使用、修改。

3. 数据泄露通知: 如发生客户数据泄露事件,甲方应在发现后【24】小时内通知乙方,并立即采取补救措施,配合乙方进行调查和处理。

4. 数据导出: 在本合同有效期内及终止后【30】日内,乙方有权随时通过平台功能或要求甲方协助,以通用格式(如CSV、JSON)导出其全部客户数据,甲方不得收取额外费用。

5. 数据清除: 在本合同终止后【30】日内,甲方应根据乙方的要求,从其服务器及所有备份系统中永久、不可恢复地删除所有客户数据,并向乙方出具书面确认函。
```

### 2.4 竞业限制条款

#### 适用场景: 劳动合同、服务合同、保密协议等

```
第X条 竞业限制

1. 竞业限制义务: 乙方(员工/服务方)同意,在本合同履行期间及终止后【二】年内,未经甲方事先书面同意,不得:
   (1) 直接或间接地从事与甲方相同或类似的业务;
   (2) 在与甲方有竞争关系的任何企业中任职、投资或提供服务;
   (3) 诱使或协助甲方的任何员工、客户、供应商终止与甲方的关系。

2. 竞业限制补偿: 在竞业限制期间,甲方应按月向乙方支付竞业限制补偿金,标准为【乙方离职前12个月平均工资的30%】。如甲方未按时支付补偿金,乙方有权解除竞业限制义务。

3. 违约责任: 如乙方违反竞业限制义务,应向甲方支付违约金人民币【金额】元,并继续履行竞业限制义务。违约金不足以弥补甲方损失的,乙方还应赔偿超出部分的损失。
```

### 2.5 里程碑与阶段验收条款

#### 适用场景: 技术开发、咨询顾问、外包、建设工程设计与施工等

```
第X条 里程碑与阶段验收

1. 项目按附件《里程碑计划》分为【立项/需求确认/样机或初稿/测试或试运行/最终交付】等阶段推进。

2. 每一阶段的工作内容、交付物、完成标准、提交时间和责任人，均以《里程碑计划》及对应阶段确认单为准。

3. 上一阶段未完成验收或未形成书面确认的，原则上不进入下一阶段；确需继续推进的，应同步确认风险、顺延安排及费用调整。

4. 阶段验收通过后，对应价款、里程碑费用或下一阶段启动条件按本合同约定执行。
```

### 2.6 交割条件与交割清单条款

#### 适用场景: 投资、股权转让、资产收购、房地产转让、复杂买卖

```
第X条 交割条件与交割清单

1. 交割前应完成的条件、需提交的文件、需办理的审批/登记事项，以附件《交割清单》逐项列明。

2. 任一项交割条件完成后，责任方应向对方提交证明文件；双方确认无误后，再进入对应付款、过户、放款或资料移交环节。

3. 交割日当日应同步办理的事项，包括但不限于价款支付、股权或资产文件签署、证照与账户移交、印章与资料交付，由双方按《交割清单》逐项勾稽确认。

4. 如交割后发现某项前置文件不真实、不完整或存在重大遗漏，守约方有权要求补正、暂停后续义务或按约追究违约责任。
```
