# 01-03 写作习惯与格式化体系

## 总原则

- 不写空话，不写正确的废话，不堆砌辞藻
- 强调信息密度

## 结构

- 先结论 → 再分析 → 再依据 → 最后策略

## 法律写作

- 不要学生腔，要像真实律师文书
- 兼顾法官阅读习惯，兼顾实务推进

## 长文

- 不要碎片化，更接近论文式连续表达
- 逻辑连续，减少机械标题感

## 笔记格式化体系

> 2026-06-02 建立。用于法律实务类笔记的视觉分层。

### 配色语义表

| 元素 | 样式 | 含义 |
|------|------|------|
| 法条名称+条文号 | `<span style="color:#C41E3A; font-family: KaiTi, STKaiti, 楷体, serif;">` 红楷体 | 法律渊源锚点——所有法条引用必须用此样式 |
| 段内子标题/关键概念 | `<span style="color:#1a6e9e; font-weight:bold;">` 深蓝加粗 | 实务操作条目、审查要点名 |
| 关键词高亮 | `<span style="background:#fff3cd; padding:1px 3px;">` 黄底 | 核心术语、系统名、触发条件 |
| 高危结论/风险提示 | `<span style="color:#B36B00; font-weight:bold;">` 橙棕加粗 | 风险结论、禁止性提示；**红色不得用于非法条内容** |
| 合规/正向 | `<span style="color:#2b6e2b; font-weight:bold;">` 绿加粗 | 合规通过标记、最佳实践 |
| 罚款金额 | `<span style="color:#B36B00; font-weight:bold;">` 橙棕加粗 | 表格中的处罚金额数字 |

### 文字修饰语义表

| 修饰 | HTML/用法 | 含义 |
|------|-----------|------|
| 加粗 | `**text**` | 强调关键结论、义务类型名 |
| 下划线 | `<u>text</u>` | **绝对红线**——不可做、不可信的硬性禁止事项 |
| 斜体 | `<em>text</em>` 或 `*text*` | 法律命题表述、区分性解释、场景化举例、背景铺垫 |
| 黄底荧光 | `<span style="background:#fff3cd;">` | 需要一眼锚定的核心术语、系统/平台名 |

### Callout 语义表

| Callout | 用途 |
|---------|------|
| `> [!important]` | 重点审核板块、核心结论 |
| `> [!warning]` | 风险提示、合规底线 |
| `> [!tip]` | 实务总结、操作建议 |
| `> [!law]-` | 超过3行的法条全文，默认折叠 |

### 禁止使用的格式

- 不用 emoji 标题（除非法律案例标记 🏛️）
- 不用 `<kbd>` 或 `<code>` 标记法律术语
- 不用彩色背景大块段落（仅限 inline 关键词高亮）
- 不用多种颜色混在同一 span 中（一个 span 只做一件事）
- **红色专用规则：红色仅用于法律、法条、法源锚点**；风险结论、禁止性提示、罚款金额等一律使用橙棕/蓝/黄/绿，不使用红色

**适用条件：** 格式化仅施用于法律实务类笔记（合规手册、案例整理、合同审查清单等）。原始协议全文、法条原文等引用材料不施加格式化。

**默认执行规则：** 法律实务类笔记的"整理"默认包含两件事：① structure cleanup（标题层级、表格、callout、章节归位）；② visual signals（红楷体法条、蓝粗概念、黄底关键词、橙棕风险、绿粗合规路径、下划线红线、斜体区分/举例）。不得只做 heading/table cleanup 后就结束。
