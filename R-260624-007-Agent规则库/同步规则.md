# 同步规则

## 架构

```
AGENT仓库（图书馆/source of truth）
    ↕ 12h 双向同步
Qoder memory（session cache/自动加载层）
```

- **AGENT仓库**：完整、结构化、人可读，编号体系永久不变
- **Qoder memory**：轻量、自动加载到session上下文，受Qoder分类体系限制

## 同步触发

**会话驱动，12h检查制：**
- 每次新会话启动时，检查距上次同步时间
- 超过12h → 执行完整双向同步
- 未超过 → 跳过，正常执行任务

## 同步方向

### Qoder → AGENT仓库
当Qoder memory中发生create/update/delete时：
1. 定位对应编号文件（通过00-索引.md中的category映射）
2. 将变更内容写入对应文件
3. 更新文件的updated日期
4. 更新00-索引.md的同步状态

### AGENT仓库 → Qoder memory
当AGENT仓库中规则/记忆发生变更时：
1. 定位对应的Qoder memory category
2. 通过UpdateMemory工具同步写入
3. 更新00-索引.md的同步状态

## 冲突处理

- 时间戳优先：以updated日期较新的版本为准
- 人工仲裁：当两边内容差异过大无法自动合并时，标注冲突并提示用户决定
- 元文档优先级最高：当元文档、AGENT仓库、Qoder memory三者内容不一致时，以元文档（最终指向AGENT仓库）为准

- 初始化日期：2026-06-24
- 初始化操作：将现有Qoder memory全量写入AGENT仓库 03-记忆库/
- 初始化后首次同步检查：下次会话启动时
